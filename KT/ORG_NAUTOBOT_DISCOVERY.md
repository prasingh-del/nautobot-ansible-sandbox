## Automation Ideas

- Generate interface configs from Nautobot data.
- Report devices missing primary IPs.
- Report network interfaces missing descriptions.
- Generate intended config for Juniper MX devices.
- Generate intended config for Cisco switches.
- Compare Nautobot intended state vs device running config.
- Build pre-change validation before any config push.

## Safe Verification Framework

Before automating any change:
1. Baseline: capture current Nautobot data and current device config.
2. Dry run: generate intended config locally only.
3. Diff: compare intended config vs current config.
4. Review: confirm with network team.
5. Apply only on test device first.
6. Validate after change.
7. Keep rollback command ready.
